## 📝 Sobre o Projeto

Este projeto foi criado com o objetivo de modernizar e automatizar o processo de reserva de produtos que estejam temporariamente fora de estoque ou que não façam parte do portfólio regular da loja, mas que possam ser encomendados para atender às necessidades do cliente.

A solução propõe a substituição dos controles manuais tradicionais por um sistema mais eficiente, proporcionando maior agilidade no atendimento, melhor organização da fila de espera e mais precisão no gerenciamento das solicitações. Com isso, a loja ganha em produtividade, reduz erros operacionais e oferece uma experiência mais satisfatória ao cliente.

## 🚀 Tecnologias Utilizadas

- **Backend:** Java
- **Frontend:** HTML, CSS e JavaScript

## 🗺️ Próximos Passos (Roadmap)

O sistema está em evolução contínua. As próximas atualizações focam em escalabilidade e melhoria na comunicação com o cliente:

- [ ] **Integração com Banco de Dados:** Substituir o armazenamento em memória/arquivos por um banco de dados relacional para gerenciar as reservas e os dados dos clientes com segurança.
- [ ] **Evolução da Arquitetura:** Refatorar a aplicação para se tornar uma **API REST** estruturada ou um projeto **Java Web** completo.
- [ ] **Notificações por E-mail:** Implementar um serviço de mensageria para enviar um e-mail automático ao cliente assim que o produto reservado chegar ao estoque, contendo as instruções para finalizar o pedido.
- [ ] **Integração com WhatsApp:** Explorar o uso de APIs de comunicação para que o cliente receba o aviso de disponibilidade do produto diretamente no seu WhatsApp, oferecendo uma experiência de atendimento mais rápida e direta.
